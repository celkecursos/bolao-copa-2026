# TASK 11 — Deploy (1/3): Preparar a VPS e as conexões SSH

Primeira das 3 fases de deploy na **VPS da Hostinger**. Prepara o servidor (requisitos) e as conexões SSH **PC → VPS** e **VPS → GitHub**. Execute após a **Task 10** (código já na branch `main` do GitHub).

> **Credenciais:** seções **SERVIDOR / VPS** e **GIT / GITHUB** do `CREDENCIAIS.txt`
> (`VPS_SSH_USER`, `VPS_IP`, `VPS_SSH_PASSWORD`, `VPS_DOMAIN`, `VPS_LINUX_USER`, `VPS_PROJECT_PATH`, `VPS_PHP_BIN`, `SSH_KEY_EMAIL`, `REPOSITORIO_GIT_SSH`, `DEPLOY_BRANCH`).
> ⚠️ Estas credenciais são de **produção** — não confundir com as de desenvolvimento.

---

## 1. Conectar o PC ao servidor com SSH

Criar a chave SSH (pública e privada) no PC, usando `SSH_KEY_EMAIL`:
```
ssh-keygen -t rsa -b 4096 -C "cesar@celke.com.br"
```
Local onde a chave é criada (Windows):
```
C:\Users\SeuUsuario\.ssh\
```
Exibir o conteúdo da chave pública:
```
cat ~/.ssh/id_rsa.pub
```

Adicionar a chave pública à VPS — **duas formas** (use qualquer uma):

**Forma A — via hPanel da Hostinger** (recomendado):
- hPanel → VPS → SSH Keys → Add SSH Key → cole o conteúdo de `id_rsa.pub`

**Forma B — via terminal** (se já tiver acesso com senha):
```
ssh root@76.13.239.186
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "CONTEÚDO_DO_id_rsa.pub" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

Testar a conexão (deve conectar sem pedir senha, apenas a passphrase da chave se houver):
```
ssh root@76.13.239.186
```

> 💡 **VS Code:** instale a extensão *Remote – SSH*, adicione o host `root@76.13.239.186` e conecte pelo painel SSH. Facilita a edição de arquivos na VPS.

## 2. Instalar os requisitos na VPS

Confira/instale (use **PHP 8.3** e **Node.js 24** — as MESMAS versões usadas para criar o projeto localmente):
- **PHP 8.3** — `php8.3 -v`
- **MySQL 8.0+** — `mysql --version`
- **Composer** — `composer --version`
- **Git** — `git -v`
- **Node.js 24** — `node -v`
- **npm** — `npm -v`

**Instalar o Node.js 24 SOMENTE se necessário:**
```
sudo apt update
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs
```
Confirme depois:
```
node -v
npm -v
```

Instalar o **Supervisor** (usado na Task 13 para manter a fila rodando):
```
sudo apt update
sudo apt install -y supervisor
```

---

## 3. Conectar a VPS ao GitHub via SSH

Gerar a chave SSH **no servidor** (`SSH_KEY_EMAIL`):
```
ssh-keygen -t rsa -b 4096 -C "cesar@celke.com.br"
```
Imprimir a chave pública gerada:
```
cat ~/.ssh/id_rsa.pub
```
- No GitHub: **Settings** do repositório → **Deploy keys** → **Add deploy key** → cole a chave pública e marque *Allow write access* → salve.

Testar a conexão com o GitHub:
```
ssh -T git@github.com
```
Se aparecer "The authenticity of host 'github.com ...' can't be established", digite `yes`:
```
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
```
✅ Sucesso esperado: `Hi <usuario>! You've successfully authenticated, but GitHub does not provide shell access.`
