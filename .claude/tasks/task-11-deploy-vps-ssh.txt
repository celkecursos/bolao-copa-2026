# TASK 10 — Deploy (1/3): Preparar a VPS e as conexões SSH

Primeira das 3 fases de deploy na **VPS da Hostinger**. Aqui preparamos o servidor (requisitos) e as conexões SSH **PC → VPS** e **VPS → GitHub**. Execute após a **Task 9** (código já na branch `main` do GitHub).

> **Credenciais:** seções **SERVIDOR / VPS** e **GIT / GITHUB** do `CREDENCIAIS.txt`
> (`VPS_SSH_USER`, `VPS_IP`, `VPS_SSH_PASSWORD`, `VPS_DOMAIN`, `VPS_LINUX_USER`, `VPS_PROJECT_PATH`, `VPS_PHP_BIN`, `SSH_KEY_EMAIL`, `REPOSITORIO_GIT_SSH`, `DEPLOY_BRANCH`).
> ⚠️ Estas credenciais são de **produção** — não confundir com as de desenvolvimento.

---

## 1. Conectar o PC ao servidor com SSH

Criar a chave SSH (pública e privada) no PC, usando `SSH_KEY_EMAIL`:
```
ssh-keygen -t rsa -b 4096 -C "cesar@celke.com.br"
```
Local onde a chave é criada (Windows):
```
C:\Users\SeuUsuario\.ssh\
```
Exibir o conteúdo da chave pública:
```
cat ~/.ssh/id_rsa.pub
```
Acessar o servidor com SSH (`VPS_SSH_USER`@`VPS_IP`), informando a `VPS_SSH_PASSWORD` quando pedido:
```
ssh root@76.13.239.186
```
Digite `yes` quando solicitado:
```
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
```
Acessar o diretório do projeto (`VPS_PROJECT_PATH`) e listar:
```
cd /home/celke-copa/htdocs/copa.celke.uno
ls
```
Se o diretório já tiver arquivos de um teste anterior, limpar (cuidado — apaga tudo, inclusive ocultos):
```
rm -rf /home/celke-copa/htdocs/copa.celke.uno/{*,.*}
```

## 2. Instalar os requisitos na VPS

Confira/instale (use **PHP 8.3** e **Node.js 24** — as MESMAS versões usadas para criar o projeto localmente):
- **PHP 8.3** — `php -v`
- **MySQL 8.0+** — `mysql --version`
- **Composer** — `composer --version`
- **Git** — `git -v`
- **Node.js 24** — `node -v`
- **npm** — `npm -v` (vem junto com o Node.js; usado na Task 11 para `npm install`/`npm run build`)

**Instalar o Node.js 24 SOMENTE se necessário** (se `node -v`/`npm -v` falharem ou `node -v` não retornar uma versão `24.x`). O **npm é instalado automaticamente** junto com o Node.js:
```
sudo apt update
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs
```
Confirme depois da instalação:
```
node -v
npm -v
```
Instalar o **Supervisor** (usado na Task 12 para manter a fila rodando):
```
sudo apt update
sudo apt install -y supervisor
```

---

## 3. Conectar a VPS ao GitHub via SSH

Gerar a chave SSH **no servidor** (`SSH_KEY_EMAIL`):
```
ssh-keygen -t rsa -b 4096 -C "cesar@celke.com.br"
```
Imprimir a chave pública gerada:
```
cat ~/.ssh/id_rsa.pub
```
- No GitHub: **Settings** do repositório (Deploy keys) ou da conta (**SSH and GPG keys**) → **New SSH key** → cole a chave pública e salve.

Testar a conexão com o GitHub:
```
ssh -T git@github.com
```
Se aparecer "The authenticity of host 'github.com ...' can't be established", digite `yes` para gravar no `known_hosts`:
```
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
```
Testar novamente:
```
ssh -T git@github.com
```
✅ Sucesso esperado: `Hi <usuario>! You've successfully authenticated, but GitHub does not provide shell access.`
