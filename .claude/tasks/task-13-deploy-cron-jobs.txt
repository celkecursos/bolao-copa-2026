# TASK 13 — Deploy (3/3): Cron, Filas (Jobs) e Atualizações

Última fase de deploy. Ativa o **agendador (cron)** — que roda a sincronização de resultados (API) e os lembretes por e-mail — e o **worker de filas** (Supervisor) para os Jobs de pontuação/e-mail. Inclui a rotina de **atualização** do projeto. Requer a **Task 12** concluída.

> **Credenciais:** `VPS_PROJECT_PATH`, `VPS_PHP_BIN`, `VPS_LINUX_USER`, `VPS_IP`, `VPS_SSH_USER` do `CREDENCIAIS.txt`.

---

## 1. Cron (agendador do Laravel)

O Scheduler executa o `worldcup:sync-matches` (sincroniza resultados reais) e os lembretes de palpite. Ver os agendamentos ativos:
```
/usr/bin/php8.3 artisan schedule:list
```

Adicionar ao crontab (rodar o scheduler a cada minuto):
```
crontab -e
```
Adicionar a linha abaixo — use `VPS_PHP_BIN` e `VPS_PROJECT_PATH`:
```
* * * * * cd /home/celke-copa/htdocs/copa.celke.uno && /usr/bin/php8.3 artisan schedule:run >> /dev/null 2>&1
```

Verificar o crontab salvo:
```
crontab -l
```

Referência do formato cron:
```
* * * * * comando
│ │ │ │ │
│ │ │ │ └── Dia da semana (0-7)
│ │ │ └──── Mês (1-12)
│ │ └────── Dia do mês (1-31)
│ └──────── Hora (0-23)
└────────── Minuto (0-59)
```

## 2. Filas (Jobs) com Supervisor

Garanta `QUEUE_CONNECTION=database` no `.env` (já definido na Task 12). Os Jobs de recálculo de pontuação (Task 6) e e-mails (Task 7) rodam pela fila.

Criar o arquivo de log antes (evita erro de spawn no Supervisor):
```
mkdir -p /home/celke-copa/htdocs/copa.celke.uno/storage/logs
touch /home/celke-copa/htdocs/copa.celke.uno/storage/logs/laravel-worker.log
chown celke-copa:celke-copa /home/celke-copa/htdocs/copa.celke.uno/storage/logs/laravel-worker.log
```

Criar o arquivo de configuração do Supervisor:
```
nano /etc/supervisor/conf.d/laravel-worker.conf
```

Conteúdo (ajuste `VPS_PHP_BIN`, `VPS_PROJECT_PATH` e `VPS_LINUX_USER`):
```
[program:laravel-worker]
process_name=%(program_name)s_%(process_num)02d
command=/usr/bin/php8.3 /home/celke-copa/htdocs/copa.celke.uno/artisan queue:work --sleep=3 --tries=3
autostart=true
autorestart=true
user=celke-copa
numprocs=1
redirect_stderr=true
stdout_logfile=/home/celke-copa/htdocs/copa.celke.uno/storage/logs/laravel-worker.log
```

Iniciar e aplicar a configuração:
```
systemctl enable supervisor
systemctl start supervisor
supervisorctl reread
supervisorctl update
supervisorctl start laravel-worker:*
```

Verificar status:
```
supervisorctl status
```
✅ Esperado: `laravel-worker:laravel-worker_00   RUNNING   pid XXXXX, uptime 0:00:XX`


## 4. Rotina de atualização (enviar nova versão para produção)

No PC (mesclar develop → main e publicar):
```
git switch main
git merge develop
git push origin main
```

Na VPS:
```
ssh root@76.13.239.186
cd /home/celke-copa/htdocs/copa.celke.uno
git pull
composer install --no-dev --optimize-autoloader
npm install
npm run build
chown -R celke-copa:celke-copa /home/celke-copa/htdocs/copa.celke.uno
/usr/bin/php8.3 artisan migrate --force
/usr/bin/php8.3 artisan optimize:clear
/usr/bin/php8.3 artisan config:cache
/usr/bin/php8.3 artisan route:cache
/usr/bin/php8.3 artisan view:cache
/usr/bin/php8.3 artisan queue:restart
systemctl restart nginx
```

> ⚠️ **Não execute `db:seed` na rotina de atualização** — o seed de produção (`ProductionSeeder`) só deve rodar uma vez na instalação inicial. Em atualizações, use apenas `migrate --force`.
